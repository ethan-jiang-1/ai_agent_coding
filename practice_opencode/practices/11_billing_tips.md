# 11 成本与额度：先靠模型、阶段和统计收住成本

## 核心习惯

OpenCode 当前官方 docs 明确给你的，不是一个很强的“单次预算硬护栏”，而是模型选择、阶段拆分和 `stats` 统计。成本控制要先从这些地方下手。

## 在 OpenCode 里怎么做

- 方向不明时，先用 `plan` agent。
- 复杂任务才上更强模型。
- 轻量内部任务交给 `small_model`。
- 定期看 `opencode stats`，不要只凭感觉估成本。
- 自动化脚本也别默认跑最贵模型。
- 交互里需要补 provider 或选模型时，用 `/connect` 和 `/models`，不要带着半配置状态盲跑。
- 如果用了 subtask、多次 `attach` 或跨客户端接力，要把重复轮次和重复调用一起算进成本。

最实用的成本控制顺序：

1. 先规划
2. 审查方向
3. 再执行
4. 用 stats 回看成本

## 这章在 OpenCode 里的现实

- 有 `opencode stats`
- 有 `opencode models --verbose`
- 有 `model` / `small_model`
- 但这次官方 docs 范围里，没有看到像 `--max-budget-usd` 这样的预算上限参数

所以别把别家工具的预算护栏想当然搬过来。

## 不要这样做

- 不要在方向还没看清时就直接高成本执行。
- 不要把所有任务都丢给最强模型。
- 不要在没有统计回看的情况下，以为“应该没花多少”。
- 不要把 `serve` / `attach` 当成预算护栏；它更像减少冷启动和上下文重建。

## 验收标准

- 你知道当前主要成本来自哪个阶段。
- 当前模型选择和任务价值匹配。
- 你有用 `stats` 或模型清单回看成本，而不是只靠感觉。
- 你能说清成本是被模型、轮次，还是 subtask / attach 协调方式放大的。
