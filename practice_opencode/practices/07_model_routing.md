# 07 模型路由：主模型做难题，小模型做轻活

## 核心习惯

OpenCode 的模型路由不是“永远上最强”，而是让强模型承担跨文件、强推理任务，让轻模型和轻量 agent 处理便宜且低风险的部分。

## 在 OpenCode 里怎么做

- 跨模块分析、复杂重构、重要审查，用更强的主模型。
- 常规实现和稳定迭代，用主力开发模型。
- 轻量内部任务让 `small_model` 去承担。
- 有明显角色差异时，按 agent 配不同模型，而不是整个项目只设一个模型。

一个实用判断：

- 需要跨文件理解吗
- 需要业务推理吗
- 需要高正确率吗

如果三项里有两项以上是“要”，就不要为了省一点成本把任务压给轻模型。

## OpenCode 里值得注意的点

- `/models` 和 `opencode models` 是正式入口
- `/models` 更偏会话内查看和切换，`opencode models` 更偏外部枚举
- `model` 是主模型
- `small_model` 只负责轻量内部任务，不等于替代主编码模型
- variants 是 OpenCode 的正式概念

## 不要这样做

- 不要把 `small_model` 当成主执行模型。
- 不要把 `/thinking` 误解成模型切换。
- 不要把所有任务都固定绑在一个模型上，不看任务难度和 agent 角色。

## 验收标准

- 当前模型选择和任务复杂度匹配。
- 强模型主要承担推理重任务。
- 轻模型只承担低风险、轻量任务。
