# 指南规划：AI Coding 高阶使用者的上下文工程实践

## 目标读者定位

已经会用 AI Coding 工具（Cursor、Claude Code、Aider、Cline、Windsurf、Trae、Gemini CLI 等），
知道如何提 Prompt，但不清楚为什么有时候 AI 越用越蠢、越来越贵的开发者。
不讲入门，直接从"会用"升级到"用好"。

---

## 核心设计原则

**每章是一个"可扩展的论点框架"，而非纯粹的建议列表。**

- 核心习惯 = 论点，必须有真实场景 / 工具真实行为 / 可量化数据作为支撑
- 每章读完，下一个人可以拿着这章继续挖：去工具文档验证、找用户案例、实测成本
- "正确做法"要有具体工具的具体操作，不能泛泛而谈
- "错误的可能"要有真实案例或可验证的工具行为
- 每章结尾标出"延伸调查方向"——哪些场景还值得深挖

---

## 每章统一格式

1. **习惯**（一句话定义）
2. **正确的做法**（具体操作，覆盖至少 3 种工具）
3. **错误的可能**（有真实案例支撑，可验证）
4. **背后的原理**（深入浅出，有比喻，可往下挖）
5. **延伸的习惯**（相关联的进阶做法）
6. **个人 / 协作注意**（仅在有差异时写）
7. **延伸调查方向**（这章还可以往哪里深挖，留给下一轮）

## 各章核心论点与可挖掘场景

### 第一章：会话生命周期——任务边界即会话边界
**论点**：一个逻辑任务完成后，立即开新会话。
**可挖掘场景**：在 50 个文件的项目中，一个简单问题拉取 18,000 Token 无关代码；Context Rot 在不同工具中的具体表现；连续三次纠错失败的会话 vs 重置后第一轮成功的对比数据

### 第二章：三次纠错法则——修改原提示词，而非追加对话
**论点**：AI 答错了，回退编辑原提示词重新运行，不要追加纠正。
**可挖掘场景**：负向 Few-shot 的具体机制；Cursor 的 Edit & Rerun 功能；追加纠错 vs 重新提示的输出质量对比实验

### 第三章：探索与执行强制分离——Plan Mode 必用
**论点**：第一轮只探索规划，禁止直接写代码。
**可挖掘场景**：Claude Code Plan Mode 的具体行为；AI 在没有 Plan 阶段直接重构时"修到第 5 个文件忘了第 1 个类型定义"的失败案例；Trae Builder Mode 的分阶段设计

### 第四章：规则文件的分层设计——轻全局，精按需
**论点**：CLAUDE.md 限 800 Token，复杂规则按路径 Glob 分层，工作流封装成 Skill 按需调用。
**可挖掘场景**：11,000 Token 规则文件 vs 800 Token 精简版的实际效果；.mdc 动态加载 vs alwaysApply 的 Token 成本差异；rulesync 工具的跨平台同步实践

### 第五章：渐进式信息投喂——@ 引用替代全文粘贴
**论点**：外部材料用动态指针挂载，不复制粘贴整篇文档。
**可挖掘场景**：API 集成时 AI 通过 Web Search 抓到过时文档的具体失败案例；llms.txt 协议的实际节省效果；MCP 接入 OpenAPI 规范后的集成质量对比

### 第六章：KV Cache 经济学——让稳定上下文摊薄成本
**论点**：稳定内容置顶，利用提示词缓存让"固定成本"零边际成本复用。
**可挖掘场景**：Anthropic 缓存命中价格（降至 10%）vs 未命中的成本对比；Gemini 默认 1024 Token 自动缓存行为；Aider --read 的缓存利用实测；一个 CLAUDE.md 每天被团队缓存命中的次数计算

### 第七章：模型路由分级——按任务复杂度选择模型
**论点**：重任务用旗舰，日常实现用 Sonnet，文件探测用 Haiku，不一视同仁。
**可挖掘场景**：推理 Token 的计费倍增机制（2-50x）；Claude Code settings.json 子智能体模型配置的实测节省比例；Cursor 用 Sonnet 连续高强度工作 8 小时不触发限制的用户报告

### 第八章：工具权限克制——联网与 MCP 按需开关
**论点**：默认关闭联网搜索，只在明确诊断出本地知识盲区时短暂开启。
**可挖掘场景**：AutoGPT 无限循环案例；AI 通过 Web Search 生成不存在 API 接口的实际案例；Gemini CLI 搜索配额超额 $14/1000 次的成本计算；ReAct 架构每次工具调用触发完整推理的机制

### 第九章：跨会话状态传递——Memento 工作流
**论点**：会话达 70% 时主动生成状态快照，新会话先读快照再工作。
**可挖掘场景**：50,000 Token 试错对话被压缩成几百词的实际效果；Cline Memory Bank 架构的具体文件结构；Claude Code auto-memory 的存储机制和读取行为

### 第十章：多智能体分流——Writer / Reviewer 双路并发
**论点**：实施者和审查者用两个独立上下文，审查者的干净上下文是发现错误的关键。
**可挖掘场景**：Git Worktree 物理隔离的具体配置；Session B 因无历史包袱发现 Session A 遗漏的命名空间冲突的案例；Trae 并行任务陷阱（十几个 Agent 同时走错方向）

### 第十一章：订阅额度陷阱——读懂你在为什么付钱
**论点**：各平台计费机制有隐性乘数，不理解就会莫名烧钱。
**可挖掘场景**：Cursor Max Mode 20% 加价机制；Windsurf 每次工具调用扣 Prompt 额度的具体数字；Claude 高峰时段额度加速消耗的用户测试报告；GitHub Copilot GPT-4.5 的 50x 倍率 vs Gemini Flash 的 0.25x

## 写作注意事项

- **比喻优先**：每个原理部分必须有比喻（"上下文窗口是 RAM 不是硬盘"）
- **数字可查**：所有量化数据标明来源工具或用户报告，让读者可以自行验证
- **工具平衡**：每章正确做法覆盖 Claude Code + Cursor + 至少一个开源工具
- **场景第一**：说服力来自场景，不是空讲原理
- **留白供挖**：每章"延伸调查方向"要给出足够具体的问题，而不是"可以继续研究"这类废话
