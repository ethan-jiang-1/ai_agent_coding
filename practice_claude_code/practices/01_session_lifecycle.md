# 01 会话生命周期：任务边界即会话边界

## 核心习惯

在 Claude Code 里，先选对会话延续方式，再开始任务。不要把“继续原任务”“从旧任务分叉试方案”“完全新开一条”混在一起。

## 在 Claude Code 里怎么做

- 新任务，直接 `claude` 开新会话。
- 只是继续最近一次同一逻辑任务，用 `claude --continue`。
- 要恢复指定历史，用 `claude --resume <session-id>`。
- 想保留历史但走另一条路径，用 `claude --resume <id> --fork-session`。

一个实用判断：

- 目标没变，只是接着做，用 `--continue` / `--resume`
- 目标相近但想试另一方案，用 `--fork-session`
- 任务边界变了，直接新开，不要硬续

## 什么时候该重开

出现下面这些信号时，不要硬续旧会话：

- 任务已经从“分析”变成“实施”
- AI 连续复读你刚否定的方向
- 会话里混入大量无关模块历史
- 当前上下文已经被试错过程污染

## 不要这样做

- 不要把 `--fork-session` 当成“干净重试”。
- 不要把所有任务都塞进 `--continue`。
- 不要忘了 Claude Code 默认会保存会话，旧历史是会回来的。

## 验收标准

- 当前会话只服务一个清晰任务边界。
- 你能解释为什么这次是 `--continue`、`--resume`、`--fork-session` 或全新会话。
- 如果别人接手，他能理解这条会话链是怎么切开的。
