# 05 渐进式信息投喂：先给锚点，再补材料

## 核心习惯

Claude Code 虽然能自己读文件，但不代表你应该一开始把所有背景、日志、文档和想法一次灌满。

## 在 Claude Code 里怎么做

- 第一轮先给任务目标、关键约束和 2-5 个锚点文件。
- 第二轮再补日志、测试输出、文档。
- 复杂任务把中间结论外化成文件，而不是一直依赖会话记忆。

Claude Code 擅长按需读文件，但它仍然需要你给出清晰边界：

- 先读哪些
- 先做分析还是先做修改
- 哪些材料只是参考

## 典型节奏

1. 任务目标 + 关键约束
2. 指定先看哪些文件
3. 补日志 / 测试 / diff
4. 如果任务分步，生成摘要或计划文件
5. 下一步基于文件继续

## 不要这样做

- 不要一开始贴几十段日志和十几个文件内容。
- 不要让所有中间结论只留在会话里。
- 不要把“Claude 能自己找”理解成“你不用给边界”。

## 验收标准

- 第一轮信息足够清晰，但不是信息大杂烩。
- 后续补充都是按需要增加，而不是一开始塞满。
- 中间关键结论已经部分外化成文件。
