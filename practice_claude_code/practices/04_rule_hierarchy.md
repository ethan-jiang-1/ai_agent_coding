# 04 规则分层：轻全局，精按需

## 核心习惯

Claude Code 的规则体系目标不是“写得越全越好”，而是让真正相关的规则在真正需要的时候进入上下文。

## 在 Claude Code 里怎么做

- 把项目级长期规则放进 `CLAUDE.md`。
- 把路径相关的规则放进 `.claude/rules/*.md`。
- 把本地个人覆盖放到 `CLAUDE.local.md`，不要污染团队规则。

实用理解：

- `CLAUDE.md` 负责慢变、跨任务的共识
- `.claude/rules/*.md` 负责路径敏感的专项规则
- `CLAUDE.local.md` 负责只对你本机有效的偏好

## 该放进规则层的内容

- 稳定的测试和验证要求
- 构建命令、包管理器约束
- 长期存在的代码风格和协作规则
- 明确不能碰的目录或文件

## 不该放进规则层的内容

- 这次任务的实施计划
- 临时调试过程
- 本周这个 feature 的拆分步骤
- 大段背景材料

## 不要这样做

- 不要把快变任务写进 `CLAUDE.md`。
- 不要把所有规则都堆在一份根文件里。
- 不要把个人偏好混进团队版本控制。

## 验收标准

- 根规则短、稳、跨任务有效。
- 路径相关规则靠 `.claude/rules/` 解决。
- 临时任务状态已经落到计划文件或 handoff 文件，而不是污染规则层。
