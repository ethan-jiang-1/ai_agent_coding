# 07 模型路由：强模型做重活，轻模型做轻活

## 核心习惯

Claude Code 的模型切换是手动的，不像 Aider 那样有内建三模型分工。所以关键不是“永远上最强”，而是让强模型只做真的难的任务。

## 在 Claude Code 里怎么做

- 跨模块分析、复杂重构、架构判断，用 `opus` 或更强模型。
- 常规实现、修 bug、文件级修改，用 `sonnet`。
- 简单格式化、轻量整理、低风险小任务，才考虑 `haiku`。

一个实用判断：

- 需要跨文件理解吗
- 需要业务推理吗
- 需要高正确率吗

如果三项里有两项以上是“要”，就不要为了省一点成本把任务压给轻模型。

## 不要这样做

- 不要把所有任务都丢给 `haiku`。
- 不要把所有任务都默认升到 `opus`。
- 不要把模型选择写成死规则，不看任务难度。

## 验收标准

- 当前模型选择和任务复杂度匹配。
- 强模型主要承担推理重任务。
- 轻模型只承担低风险、低耦合工作。
