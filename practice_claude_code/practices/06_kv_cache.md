# 06 稳定前缀：让规则稳定，让历史可压缩

## 核心习惯

对 Claude Code 来说，最值得稳定的前缀通常是 `CLAUDE.md` 和路径规则；最值得及时压缩的是不断增长的对话历史。

## 在 Claude Code 里怎么做

- 让 `CLAUDE.md` 和 `.claude/rules/` 慢变。
- 会话过长时，及时 `/compact`。
- 任务级信息不要写进规则层，而要落盘成计划文件或 handoff 文件。
- 不确定是否该压缩前，先看 `/context`，再决定继续聊、`/compact` 还是重开。

`/compact` 在 Claude Code 里的重要性有两个：

- 清理历史负担
- 重新把规则层从磁盘注入回来

## 不要这样做

- 不要把“有规则文件”理解成“长会话一定没问题”。
- 不要频繁重写 `CLAUDE.md` 去承载短期任务。
- 不要让历史无限增长然后指望上下文质量不变。

## 验收标准

- 规则文件明显慢于任务文件变化。
- 任务状态已经外化到文件，而不是都挤在历史里。
- 会话变长时，你有 `/compact` 或重开机制。
