# 07 模型路由：强模型做重活，贵模式只在该贵时用

## 核心习惯

Cursor 的模型路由主要靠手动选择或按 mode 固定，不像 Aider 那样有内建三模型分工。所以关键不是“永远上最强”，而是让贵模型和 Max Mode 只做真的难的任务。

## 在 Cursor 里怎么做

- 日常实现、常规修 bug，用默认的日常模型或 Auto。
- 跨模块分析、复杂重构、架构判断，再切到更强模型。
- 代码库很大、文档很多、上下文确实不够时，才开 Max Mode。
- 如果某类任务总是用同一种模型，可以把它固化到 `Custom Mode`。
- 当前官方 editor 文档没有把 `/model` 列成 quick command；这章主要仍靠 model picker 和 modes。

一个实用判断：

- 需要跨文件理解吗
- 需要复杂业务推理吗
- 需要更大上下文吗

如果前两项都不是“要”，通常没必要默认最贵配置。

## 不要这样做

- 不要把所有任务都默认升到最贵模型。
- 不要把 Max Mode 当成常驻默认。
- 不要把模型选择写成死规则，不看任务难度。

## 验收标准

- 当前模型选择和任务复杂度匹配。
- 强模型主要承担推理重任务。
- Max Mode 只在上下文真的不够时开启。
