# 05 `@` 引用、代码库检索与 handoff

## 常用上下文入口

Cursor 这章最重要的是“用指针代替大段粘贴”。

常见入口包括：

- `@file`
- `@folder`
- `@Past Chats`
- `@Cursor Rules`
- 代码库搜索 / docs 检索

## `@file`

适合：

- 给 Agent 一个明确锚点
- 指定参考实现
- 指定只想讨论的关键文件

注意：Cursor 往往会做相关片段截取，不一定把全文都注进上下文。

## `@folder`

适合：

- 给出模块范围
- 让 Cursor 在一个目录内扩展检索

但不要一上来就把整个仓库都丢进去。

## `@Past Chats`

适合：

- 引用旧对话里的已确认结论
- 跨天续任务时补上下文

不适合：

- 把所有旧历史都重新灌回当前任务

## 日志和测试输出

当内容是“这次运行的瞬时事实”时，通常直接粘贴或附文件更稳，例如：

- 报错堆栈
- 测试失败输出
- 构建日志

## handoff 文件模板

```text
# 当前状态
- 已完成：
- 未完成：

# 已确认约束
- ...

# 下一步
1. ...
2. ...

# 风险
- ...
```

## 这章最容易写错的地方

- 不要把 `@file` 当成“全文一定注入”。
- 不要一次把太多文件和目录都加进去。
- 不要让关键中间结论只留在 tab 历史里。
