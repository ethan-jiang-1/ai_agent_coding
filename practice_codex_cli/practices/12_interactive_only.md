# 12 交互式专属能力：那些不适合硬塞进原章节的 slash commands

## 核心定位

这一章不是新发明一个“第 12 个习惯”，而是承接 Codex CLI 交互式命令面里那些很重要、但没有合理 external 对应、也不适合硬塞进前面章节的能力。

## 这一章为什么存在

这次对 Codex CLI 做 command-surface mapping 之后，发现有一批 slash commands：

- 明显属于交互式会话
- 对真实使用体验有价值
- 但很难和现有外部命令形成清晰的一一映射

对这类能力，强行塞回前面章节只会让结构变乱，所以单独收在这里更干净。

## 当前收录的代表

- `/help`
- `/bug`
- `/doctor`
- `/terminal-setup`

这些命令更像：

- 会话内帮助入口
- 交互式故障排查入口
- 终端环境初始化或修复入口

而不是外部命令的替代品。

## 应该怎么理解它们

- 它们属于交互工作流增强层
- 它们对日常使用重要，但不是原有 11 个习惯的核心骨架
- 它们更适合在已经进入交互式 TUI 之后使用

## 不要这样做

- 不要因为它们重要，就把它们误当成新的通用习惯。
- 不要把所有没地方放的内容都往这里扔。
- 不要忽略它们和真实交互体验之间的关系。
