# 05 渐进式信息投喂：用指针和文件外化，不要一次性灌满

## 核心习惯

Codex CLI 没有 Aider 那种 repo map 自动补全机制，所以信息投喂要更主动：先给任务边界，再给关键路径，再补日志、测试、diff。

## 在 Codex CLI 里怎么做

- 第一轮只给任务目标、约束和 2-5 个锚点文件。
- 第二轮再补文档、日志、测试输出。
- 需要跨多轮传递信息时，用 `-o` 把结果写文件，再喂给下一轮。

对 Codex CLI 来说，最有用的不是“把所有内容粘进 prompt”，而是：

- 在提示词里点名路径
- 用 stdin 传完整日志或 diff
- 用文件把上一步结论显式化

## 典型节奏

1. 任务描述 + 核心约束
2. 指定先读哪些文件
3. 补测试失败或日志
4. 如果任务分步，输出摘要文件
5. 下一步读取摘要文件继续

## 不要这样做

- 不要一开始粘几十段日志和十几个文件内容。
- 不要让 AI 在一个会话里“自己记住全部中间结论”。
- 不要把引用文件和粘贴全文混为一谈。

## 验收标准

- 第一轮提示词是清晰的，不是信息大杂烩。
- 后续补充都是按需要增加，而不是一次塞满。
- 中间结论已经外化成文件，而不是只存在对话历史。
