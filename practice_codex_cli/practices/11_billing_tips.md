# 11 成本与额度：先分清你在花什么

## 核心习惯

Codex CLI 的成本边界，先取决于你怎么登录，再取决于你怎么跑任务。不要把“模型很强”和“这次就该这么花”画等号。

## 在 Codex CLI 里怎么做

- 先分清自己是走 ChatGPT 计划登录，还是走 API key / 自动生成的 API key。
- 再分清当前任务是规划、实现、还是轻量清理。
- 最后才决定模型、会话长度和自动化程度。

一个简单的成本控制顺序：

1. 读少量关键文件，先判断范围
2. 只读规划并落盘
3. 只把难任务交给强模型
4. 简单任务才降到本地模型或低成本路径

## 在 Codex CLI 里真正影响成本的事

- 会话是否过长
- 是否把错误方向跑进了可写执行阶段
- 是否把简单任务也用最强模型做
- 是否把中间结论外化，避免重复分析

## 不要这样做

- 不要把最新价格或模型列表硬编码进长期文档。
- 不要在需求还没看清的时候直接进高成本执行。
- 不要把“更强自治”误认为“天然更省钱”。

## 验收标准

- 你知道当前命令属于计划、执行还是轻量清理。
- 你知道自己是在吃计划额度还是在消耗 API 用量。
- 这次模型和运行形态的选择有明确理由，而不是默认堆最强配置。
