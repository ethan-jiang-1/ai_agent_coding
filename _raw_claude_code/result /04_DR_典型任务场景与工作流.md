# DR 子上下文 4：典型任务场景与工作流

## 研究目标

请研究：**在典型 coding agent 工作场景中，高手是如何通过上下文管理习惯来组织整条协作工作流的。**

这里的“上下文管理”主要指：

- 第一轮怎么发任务
- 中途怎么补信息
- 长会话什么时候该压缩或重开
- 什么放在聊天里，什么放在长期规则里
- 什么时候开工具，什么时候切模型

## 重点场景

- 修 bug
- 大规模重构
- 理解陌生代码库
- 写技术文档 / PRD / 设计说明
- 调 API / 查日志 / 多工具协作

## 先限定研究对象

- Claude Code
- Cursor
- Codex
- Trae
- 至少一个开源 coding agent

## 搜索词

- bugfix workflow coding agent
- refactor workflow coding agent
- codebase understanding workflow
- AI coding agent memory workflow
- long session summary coding assistant
- coding agent project rules
- coding agent tool use workflow

## 优先去哪里找

- 官方 blog 里的案例文章
- docs 里的 tutorial / quickstart / examples
- YouTube 演示、播客、公开演讲
- 高水平用户的实战复盘
- GitHub issue / discussion 里关于工作流和踩坑总结

## 每个场景重点看什么

- 低效用户怎么把会话越聊越乱
- 高效用户第一轮怎么定任务
- 中途什么时候做摘要、换会话、外置规则
- 哪些产品机制能明显帮到这个场景
