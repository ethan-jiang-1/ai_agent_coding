# DR 子上下文 1：通用上下文管理习惯

## 研究目标

请研究：**哪些上下文管理习惯几乎适用于所有 coding agent。**

先把“上下文管理”理解成用户动作，不是只盯着模型底层：

- 怎么发任务
- 怎么接着追问
- 什么时候该开新会话
- 什么时候该做摘要
- 什么时候把信息放进长期规则或项目记忆

先限定在 coding agent 范围内，优先看：

- Claude Code
- Codex
- Cursor
- Trae
- OpenCode / Aider / Continue / Cline / Roo Code / Windsurf / Gemini CLI

## 重点问题

- 用户最常见的坏习惯是什么
- 高水平用户最稳定的好习惯是什么
- 哪些习惯既省钱又减少返工
- 哪些习惯几乎不依赖特定产品

## 重点搜寻词

- coding agent habits
- context management habits
- prompt iteration vs edit
- long session summary
- new chat vs same chat
- project rules / memory
- 大材料复用 / project context / workspace memory

## 优先去哪里找

- 官方 usage guide / best practices
- 产品团队成员公开经验
- GitHub issues / discussions 里关于 memory、rules、context、session 的讨论
- 高水平用户的 workflow 分享

## 特别留意这些通用习惯

- 回答方向不对时改原任务
- 第一轮合并说明相关任务
- 长会话及时摘要或重开
- 稳定背景移出聊天框
- 大材料不要每轮重贴
- 小任务不要默认高价模型
- 工具和联网不要默认全开
