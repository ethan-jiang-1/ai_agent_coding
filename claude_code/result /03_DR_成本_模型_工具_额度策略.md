# DR 子上下文 3：成本、模型、工具与额度策略

## 研究目标

请研究：**不同 coding agent 中，哪些上下文管理习惯会直接影响成本结构，以及这些习惯如何与模型能力发挥联系起来。**

这里的“成本”不只指 API 钱，也包括：

- token 消耗
- API 费用
- 套餐额度
- 时间消耗
- 返工成本

## 先限定对象

- Claude Code
- Codex
- Cursor
- Trae
- OpenCode / Aider / Continue / Cline / Roo Code / Windsurf / Gemini CLI

## 重点问题

- 小任务为什么不应该长期使用高价模型
- 工具和联网为什么经常会带来隐性成本
- 长会话为什么会让重读历史成为主要消耗
- 大材料为什么要改成项目常驻而不是反复上传
- 某些产品的额度窗口、高峰时段、超量机制会怎样影响用户习惯

## 搜索词

- coding agent cost
- tool calling cost
- long context cost
- session compact
- model routing
- model tier
- rate limit / quota / usage limit
- token cost coding assistant

## 优先去哪里找

- pricing 文档
- usage limits / rate limits / quota 文档
- docs 里关于 tool use、search、connectors、reasoning 的说明
- 用户讨论里关于“为什么一下子用量暴涨”的复盘
- 团队成员对模型分层和使用建议的公开说明

## 特别留意的习惯

- 按任务切模型
- 非必要工具默认关闭
- 联网和搜索按需开启
- 长会话及时压缩
- 大材料常驻化
- 重任务和轻任务分流
- 按额度窗口安排关键任务
