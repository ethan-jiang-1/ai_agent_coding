# DR 子上下文 2：产品机制与上下文承载

## 研究目标

请研究：**不同 coding agent 分别用什么机制承载和管理上下文，这些机制会诱导出哪些用户习惯。**

这里的重点不是列功能，而是看：

- 用户把背景放在哪里
- 用户怎么让 agent 持续记住项目信息
- 用户怎么减少重复贴资料

## 重点对象

- Claude Code
- Codex / OpenAI Codex / ChatGPT coding workflow
- Cursor
- Trae
- OpenCode
- Aider / Continue / Cline / Roo Code / Windsurf / Gemini CLI

## 重点问题

- 它如何处理会话历史
- 它有没有长期记忆
- 它有没有项目级规则或项目说明
- 它如何处理代码库索引、知识库、RAG、文件检索
- 它是否支持会话压缩、摘要、compact
- 它是否支持规则文件、工作区记忆、项目记忆

## 搜索词

- coding agent memory
- project rules
- workspace memory
- repo instructions
- session compact
- codebase indexing
- RAG retrieval coding assistant
- CLAUDE.md / Cursor Rules / memory / instructions

## 特别去找的地方

- docs 里关于 memory、rules、instructions、workspace、project 的章节
- changelog 里新增 memory / indexing / compact 功能的说明
- GitHub repo 和 issues 里关于上下文丢失、规则文件、索引机制的讨论
- 团队成员对“为什么要这样设计”的公开解释

## 研究时特别留意

- 哪个产品把“长期背景”放在 rules / memory / project docs 里
- 哪个产品更依赖会话本身
- 哪个产品更依赖代码库索引和检索
- 这些机制分别会把用户带向什么使用习惯
