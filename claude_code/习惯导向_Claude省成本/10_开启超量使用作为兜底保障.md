# 习惯 10：开启超量使用作为兜底保障

## 原文定义

如果你的套餐支持超量使用，就把它作为兜底保障打开，避免关键时刻突然断线。

## 适用范围

- 主标签：Claude 生态
- 原因：这条直接依赖 Anthropic 的套餐与超量使用计费机制

## 为什么原文这么强调

- 当会话额度耗尽时，工作可能被直接打断
- 对连续工作场景来说，突然中断的代价很高
- 兜底开关的意义不是鼓励乱用，而是保证关键节点不断

## 场景要怎么理解

原文描述的是：

- Pro、Max 5x、Max 20x 订阅用户
- 可以在“设置”→“用量”里开启“超量使用”
- 会话额度耗尽后，转到按 API 费率的按需计费

## 顺着原文补充的理解

这条习惯优化的是“中断风险”，不是直接降低单次 token 成本。

## Claude Code 对应动作

- 在重要交付前，先确认自己有没有可承受的兜底方案
- 把它当保险，不要把它当成常态化放大消耗的理由
