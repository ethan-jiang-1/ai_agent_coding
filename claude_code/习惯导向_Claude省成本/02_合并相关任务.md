# 习惯 2：合并相关任务

## 这个习惯是什么

把彼此相关的问题尽量合并成一条消息，而不是拆成很多轮发送。

## 为什么这会省成本

- 多轮拆开发送，意味着多次加载上下文
- 目标拆碎后，模型不容易看见完整任务边界
- 同样的材料被反复读取，实际消耗会变大

## 背后的机制

Anthropic 官方 best practices 明确建议：多个相关任务应该尽量放进同一条 message。

这背后不只是省轮次，更是在减少重复读取和减少需求丢失。

## 手动对话例子

低效写法：

```text
总结这篇文章
列出三个要点
再给我一个标题
```

高效写法：

```text
请总结这篇文章，提炼三个核心要点，并给出三个适合公众号发布的标题。
```

## Claude Code 对应动作

在 Claude Code 里，最好一次交代这四类信息：

- 目标
- 约束
- 输出格式
- 验证方式

例如：

```text
请重构这个目录下的文章素材。
目标：改成习惯导向切片
约束：不改原始文件
输出：按习惯拆成多个 md
验证：目录结构清晰，文件可直接阅读
```

## 这类习惯真正优化的是什么

它优化的是任务完整度。完整度越高，追问越少，返工越少。
