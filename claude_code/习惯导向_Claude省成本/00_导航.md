# Claude 省成本的习惯切片导航

这个目录不是原文备份，而是把原始材料重新拆成“人能消化、能执行、能迁移”的习惯卡片。

组织原则有四个：

- 以用户习惯为主线，而不是以功能名堆概念
- 每张卡片都同时覆盖手动对话和 Claude Code 两种场景
- 每张卡片都补上背后的机制，不只说做法
- 每张卡片都尽量落到例子、命令和参考证据

## 关于“轮次”的统一说明

这个目录里凡是出现“低效写法 / 高效写法”“手动对话例子”时，都默认按下面这个口径理解：

- “一轮”指的是一次发送、一次提交
- 如果内容分三次发出去，那就是三轮，不管三句话长得多像一个完整段落
- 如果内容一次性发出去，即使里面有三项任务，它也仍然是一轮

所以这里讨论的，很多时候不是字数多少，而是：

- 任务有没有在第一轮一次性交代完整
- 还是靠第二轮、第三轮不断补充

你一开始要的，其实就是这种结构：

- 习惯导向
- 面向用户
- 带技术干货
- 带范例
- 有网络证据沉淀到 `reference`
- 底下补 Claude Code 可操作命令
