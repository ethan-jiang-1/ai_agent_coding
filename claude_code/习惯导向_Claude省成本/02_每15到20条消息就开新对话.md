# 习惯 2：每 15～20 条消息就开新对话

## 原文定义

当一个对话已经很长时，让 Claude 先总结，再开一个新对话，把总结作为第一条消息继续。

## 适用范围

- 主标签：通用
- Claude Code 关系：适用，而且可以直接借助 `/compact`

## 为什么原文这么强调

- 对话越长，后续每一轮重读的历史越多
- 大量 token 会花在重复读取旧消息上
- 真正新产生的有效内容，在长对话里占比会越来越低

## 轮次要怎么理解

这条习惯不是说“对话中不能有第 16 条消息”，而是说：

- 如果已经累积了很多轮
- 后面再继续接龙就越来越贵
- 这时应该用一轮摘要，替换前面很多轮松散历史

示意：

```text
第 1～18 轮：已经来回讨论很多内容
第 19 轮：请总结当前对话的已确定结论、未解决问题、当前约束和建议下一步
新会话第 1 轮：粘贴上面的总结，继续工作
```

## 顺着原文补充的理解

这条习惯优化的是“长对话的历史负担”，不是否定多轮协作本身。

## Claude Code 对应动作

- 会话变长时使用 `/compact`
- 把长期规则放进 `CLAUDE.md`，避免压缩后还要重复补背景
