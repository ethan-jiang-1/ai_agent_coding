# 习惯 10：把省成本做成工作流

## 这个习惯是什么

不要把“省 token”理解成零散技巧，而要把它编排成一套顺序稳定的工作流。

## 一套可直接执行的顺序

1. 先选模型
2. 再关掉不需要的功能
3. 用模板一次讲清任务
4. 把长期背景移出会话
5. 把大材料放进 Projects
6. 长对话及时压缩
7. 回答方向不对时先编辑原消息

## 为什么这会省成本

- 单个技巧只能局部优化
- 工作流才能稳定降低返工、重读和重复加载
- 顺序固定后，人的使用习惯才会形成

## 一个完整例子

如果你要让 Claude 帮你改一份行业分析报告，可以这样排：

```text
先用 Sonnet
关闭不需要的搜索和工具
把行业报告放进 Project
把默认输出偏好写进长期记忆
用固定模板发任务
如果方向不对，编辑原提示词
如果会话变长，用摘要或 /compact
```

## Claude Code 对应动作

- `/model`
- `/status`
- `/memory`
- `/compact`
- `/init`

## 这类习惯真正优化的是什么

它优化的是“整条协作链路”。真正成熟的使用者，不是会几个技巧，而是会编排整条链路。
