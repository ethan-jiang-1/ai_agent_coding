# 习惯 1：多去编辑你的提示词

## 原文定义

当 Claude 没有理解你的意思时，不要继续发送“不是这个意思”“再改一下”这类新消息，优先编辑原始提示词后重新生成。

## 为什么原文这么强调

- 每条新消息都会追加进对话历史
- 后续每一轮都要重新读取前面的历史
- 纠错消息越多，真正任务越容易被埋进噪音里

## 轮次要怎么理解

低效写法是多轮补丁：

```text
第 1 轮：帮我写产品介绍
第 2 轮：不对，太硬了
第 3 轮：也不对，再短一点
第 4 轮：还是不对，要面向大学生
```

更符合原文意思的做法是：

```text
编辑第 1 轮原消息：
帮我写产品介绍，面向大学生，语气轻松，控制在 120 字以内，重点突出低门槛和快速上手。
```

## 顺着原文补充的理解

这条习惯省的不是“字数”，而是少开纠错轮次。

## Claude Code 对应动作

- 方向不对时，优先回到原任务改写
- 把长期风格偏好放进 `CLAUDE.md` 或记忆，减少重复纠偏
