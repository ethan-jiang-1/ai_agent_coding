# 习惯 9：避开高峰时段

## 原文定义

资源密集型任务尽量放到非高峰时段处理，不要默认在高峰时段猛跑。

## 适用范围

- 主标签：Claude 生态
- 原因：这条直接依赖原文给出的 Anthropic 高峰时段额度策略

## 为什么原文这么强调

- 原文指出，高峰时段对 5 小时会话额度的消耗更快
- 同样的任务，放在不同时间段，额度压力可能不同
- 如果能把重任务挪到非高峰时段，套餐效率会更高

## 原文给出的高峰时段

- 工作日太平洋时间上午 5:00 至 11:00
- 北京时间晚上 8:00 至凌晨 2:00

## 顺着原文补充的理解

这条习惯优化的是“任务排程”，尤其适合长会话、重推理、大材料处理这类高消耗任务。

## Claude Code 对应动作

- 大型重构、长链路推理、长文档分析尽量避开高峰时段
- 轻任务受影响较小，优先把重任务调度出去
