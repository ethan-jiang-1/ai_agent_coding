# 习惯 7：按任务切模型

## 这个习惯是什么

不要默认永远用最强模型，而要按任务强度切模型。

## 一个够用的分层

- Haiku：轻任务
- Sonnet：主力任务
- Opus：高推理任务

## 为什么这会省成本

- 模型越强，通常单价越高
- 轻任务用高价模型，收益和成本不匹配
- 真正省成本的方式不是盲目降级，而是按任务配模型

## 背后的机制

Anthropic 官方价格表显示，Haiku 4.5、Sonnet 4.5、Opus 4.6 的输入和输出单价存在明显梯度差异。

所以模型选择本身就是一个成本开关。

## 手动对话例子

- 翻译、改写、标题生成、格式整理：优先 Haiku
- 代码理解、文章结构重构、主力协作：优先 Sonnet
- 高风险判断、复杂拆解、关键决策：再用 Opus

## Claude Code 对应动作

- 用 `/model` 切换模型
- 用 `/status` 检查当前模型

## 这类习惯真正优化的是什么

它优化的是“能力投入结构”。把高成本能力留给真正值得的任务。
