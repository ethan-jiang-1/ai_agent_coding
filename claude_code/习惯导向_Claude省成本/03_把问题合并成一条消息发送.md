# 习惯 3：把问题合并成一条消息发送

## 原文定义

多个相关任务，尽量放进一条提示词里一次发出去，不要拆成多条消息逐轮补发。

## 适用范围

- 主标签：通用
- Claude Code 关系：适用，但本质上是所有 agent 的任务布置原则

## 为什么原文这么强调

- 三个独立提示词，意味着三次上下文加载
- 一个包含三个任务的提示词，只需要一次上下文加载
- 模型在第一轮看到完整需求，结果通常也更稳

## 轮次要怎么理解

低效写法是三轮：

```text
第 1 轮：总结这篇文章
第 2 轮：现在列出要点
第 3 轮：再给我建议一个合适标题
```

高效写法是一轮：

```text
第 1 轮：对这篇文章进行总结，列出主要要点，并给一个合适的标题建议。
```

如果三句话虽然长得像一段，但发送时仍然分三次发，那它还是三轮，不是一轮。

## 顺着原文补充的理解

这条习惯优化的是“第一轮任务完整度”。

## Claude Code 对应动作

在一次任务布置里尽量交代清楚：

- 目标
- 约束
- 输出格式
- 验证方式
