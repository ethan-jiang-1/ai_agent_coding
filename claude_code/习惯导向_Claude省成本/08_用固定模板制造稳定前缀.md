# 习惯 8：用固定模板制造稳定前缀

## 这个习惯是什么

把高频任务写成模板，只替换变量部分，不要每次从零自由发挥。

## 为什么这会省成本

- 模板会让任务结构更稳定
- 稳定结构更利于复用和缓存
- 任务边界更清楚，返工更少

## 背后的机制

Anthropic 的 prompt caching 文档明确指出：

- cache hits 依赖 100% 一致的前缀
- 稳定、可复用的内容最适合缓存
- 缓存内容放在提示词前部效果最好

这背后的逻辑很简单：固定部分越稳定，系统越容易把它当成可复用前缀。

## 手动对话例子

```text
角色：编辑助手
任务：拆解文章
输出：核心观点、证据、反例、改写建议
风格：中文、短句、偏实操
材料：<<<这里替换本次文章>>>
```

## Claude Code 对应动作

- 把固定流程写进 skill 或项目规范
- 把重复性很高的说明放在 `CLAUDE.md`
- 让“规则部分固定，变量部分替换”成为默认写法

## 这类习惯真正优化的是什么

它优化的是“稳定前缀”。这就是你最早提到的 KV cache / prompt cache 在用户习惯层面的投影。
