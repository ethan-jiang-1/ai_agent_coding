# 习惯 4：把长期背景沉淀为记忆

## 这个习惯是什么

凡是你会重复讲三遍以上的内容，都不该继续放在聊天里，而要沉淀成长期记忆。

## 适合沉淀的内容

- 默认语言
- 输出结构
- 常用风格
- 项目约束
- 常用命令
- 命名规范

## 为什么这会省成本

- 同样背景每次重讲，本质上是在重复支付上下文成本
- 长期规则如果不外置，新会话每次都要重新建模
- 稳定背景越固定，越利于复用

## 背后的机制

Anthropic 的个性化文档说明，Profile preferences 是 account-wide；Claude Code 文档说明，`CLAUDE.md` 会在每个 session 开始时加载。

这说明，长期信息应该有长期载体，不应该假装放在临时会话里。

## 手动对话例子

不要反复说：

```text
默认中文，先结论后分析，尽量短句，不要太学术。
```

应该沉淀到：

- Profile preferences
- Project instructions
- `CLAUDE.md`

## Claude Code 对应动作

- `/memory`
- `/init`
- 手动维护 `CLAUDE.md`

## 这类习惯真正优化的是什么

它优化的是“跨会话的重建成本”。一次写清，多次复用。
