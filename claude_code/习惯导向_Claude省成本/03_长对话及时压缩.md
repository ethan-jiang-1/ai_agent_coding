# 习惯 3：长对话及时压缩

## 这个习惯是什么

一旦对话已经很长，就不要继续无限接龙，而是先总结，再开新会话继续。

## 为什么这会省成本

- 长对话会不断挤占上下文窗口
- 越往后走，真正有价值的信息占比通常越低
- 会话太长时，历史噪音会干扰后续判断

## 背后的机制

Anthropic 官方把 context window 描述成模型单轮的工作记忆。Claude Code 文档也说明，长会话可以用 `/compact` 做压缩，而 `CLAUDE.md` 会在压缩后重新注入。

这意味着：

- 会话信息可以压缩
- 长期规则应该外置

## 手动对话例子

```text
请把当前对话整理成：
1. 已确定结论
2. 未解决问题
3. 当前约束
4. 建议的下一步
```

整理完后，把摘要带到新会话。

## Claude Code 对应动作

- 使用 `/compact`
- 把真正长期有效的规则写进 `CLAUDE.md`

## 这类习惯真正优化的是什么

它优化的是上下文质量。不是让你丢内容，而是让保留下来的内容更有密度。
