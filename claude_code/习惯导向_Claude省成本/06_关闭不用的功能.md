# 习惯 6：关闭不用的功能

## 这个习惯是什么

搜索、连接器、工具、深度思考，不要默认全开。当前任务用不到，就先关掉。

## 为什么这会省成本

- 功能越多，不等于越划算
- 对简单任务来说，额外功能只会带来额外开销
- 很多任务只需要基础生成，不需要联网、检索或重推理

## 背后的机制

Anthropic 官方文档明确指出：tools 和 connectors 是 token-intensive。也就是说，额外能力本身会吃掉上下文预算和使用额度。

## 手动对话例子

这些场景通常不需要把能力开满：

- 改标题
- 润色文案
- 做翻译
- 调整格式
- 提炼提纲

## Claude Code 对应动作

在 Claude Code 里，先判断当前任务是否真的需要：

- 联网检索
- 外部服务
- 长链路推理

如果不需要，就让任务保持简单、直接。

## 这类习惯真正优化的是什么

它优化的是“能力与任务的匹配度”。该轻就轻，该重再重。
