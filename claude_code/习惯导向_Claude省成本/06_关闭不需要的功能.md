# 习惯 6：关闭不需要的功能

## 原文定义

网页搜索、连接器、探索模式、深度思考这类功能，不需要时就关掉，不要默认常开。

## 为什么原文这么强调

- 额外能力并不等于更划算
- 简单任务往往只需要基础生成
- 功能一旦开着，可能带来额外 token 消耗

## 场景要怎么理解

原文指向的任务很明确：

- 写自己的内容
- 改标题
- 润色文案
- 翻译
- 调整格式

这类任务通常不需要联网、连接器或额外深推理。

## 顺着原文补充的理解

这条习惯优化的是“能力开关和任务强度的匹配”。

## Claude Code 对应动作

- 开工前先判断这次任务是否真要联网
- 不需要外部能力时，就让任务保持直接、简单
