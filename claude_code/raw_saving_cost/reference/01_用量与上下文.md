# 01 用量与上下文

## 来源

- https://support.claude.com/en/articles/11647753-understanding-usage-and-length-limits
- https://support.anthropic.com/en/articles/9797557-usage-limit-best-practices
- https://support.anthropic.com/en/articles/8114491-getting-started-with-claude

## 关键摘录

### 关于用量限制

- “Your usage is affected by several factors, including the length and complexity of your conversations, the features you use, and which Claude model you're chatting with.”
- “Note that your usage of all different Claude product surfaces (claude.ai, Claude Code, Claude Desktop) counts towards the same usage limit.”

### 关于上下文窗口

- “Length limits relate to Claude's context window—the amount of information Claude can work with in a single chat.”
- “Claude's context window size is 200K tokens across all models and paid plans...”

### 关于项目与工具

- “Projects use retrieval-augmented generation (RAG), which allows Claude to work with larger amounts of information more efficiently by only loading relevant content into the context window.”
- “Tools and connectors are token-intensive, so managing them helps both maximize your available context window and optimize your usage limits.”

### 关于消息习惯

- “Take a moment to review your message for clarity and completeness to reduce the need for follow-up messages.”
- “If you have multiple related tasks or questions, group them in a single message.”

### 关于 5 小时窗口

- “While using the free Claude plan, there is a session-based usage limit that will reset every five hours.”

## 对本文可用的结论

- 对话越长、越复杂，用量越高，这支持“少发无效补充消息”和“长会话及时压缩/重开”的习惯。
- 项目机制本质上是在减少无关内容进入上下文，这支持“把大材料放进 Projects，而不是反复粘贴”。
- 工具、连接器、搜索都会吃上下文和额度，这支持“默认关闭不用的功能”。
